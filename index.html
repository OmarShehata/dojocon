What is this?

This is a handy page of resources mentioned in our DojoCon session on teaching students to code by digging through open source games.

Resources to use

There's tons of awesome tools out there. We've found these to be especially effective at 1) being easy to setup in a 2 hour sessions and 2) engaging a broad group

The Big List of Open Source Games

This is an awesome list of open source games in a variety of engines, languages and platforms. Pick one and explore it at your next Dojo!

Here's an example we did with the HTML5 Super Mario.

PlayCanvas

PlayCanvas is an awesome web-based 3D games editor. It requires no setup (beyond creating an account) and coders can jump right in. Any project on there can be forked, so it's a great source of inspiration too!

Here's an example we did for a simple Apollo 11 space ship simulator.

Mozilla Thimble

Thimble is a great tool for collaborative learning of web tools. Unlike CodePen, the project setup mirrors local development, so it bridges a knowledge gap in transitioning to bigger projects. It also highly encourages remixing.

General Tips

  • Try to start with a broken or an unfinished version of the game. It's a lot easier to avoid distraction and give an incentive to dive in the code.
  • Give the coder at least one immediate thing to try first that will yield an obvious effect. Having a place to start avoids overwhelming confusion.
  • Suggest ideas for further hacks of increasing difficulty.
